# める配くん連携WordPressプラグイン 仕様書

## プロジェクト概要

### プラグイン名
める配くん連携

### バージョン
1.0.0

### 作者
Delightful Inc.

### 目的
Contact Form 7の投稿内容をめる配くんのAPIを実行して読者登録するWordPressプラグイン

## 機能仕様

### 1. 基本機能

#### 1.1 API連携機能
- **エンドポイント**: める配くんのAPIエンドポイント
  - 開発環境: `https://XXXXX.mgzn.jp/api/apiText.php`
- **認証方式**: クライアントログインID + パスワード
- **データ形式**: プレーンテキスト（UTF-8）

#### 1.2 管理画面機能
- **設定画面**: WordPress管理画面の「設定」→「める配くん連携」
- **設定項目**:
  - エンドポイントURL
  - ログインID
  - パスワード
- **接続テスト**: API接続の確認機能
- **フィールドマッピング**: Contact Form 7フィールドとCSVレイアウトのマッピング

#### 1.3 自動読者登録機能
- **トリガー**: Contact Form 7の送信時（`wpcf7_mail_sent`フック）
- **処理内容**: マッピング設定に基づいてCSVデータを作成し、める配くんAPIに送信
- **重複チェック**: 既存読者との重複をチェック

### 2. データ処理仕様

#### 2.1 CSVヘッダー管理
- **保存タイミング**: 接続テスト成功時
- **保存場所**: WordPressオプション（`meruhai_csv_headers`）
- **用途**: 読者登録時のCSVデータ構築

#### 2.2 フィールドマッピング
- **マッピング対象**: Contact Form 7の全フィールド
- **除外フィールド**: 
  - 登録日（自動設定）
  - 未使用
  - SMSステータス
- **保存場所**: WordPressオプション（`meruhai_field_mapping`）

#### 2.3 CSVデータ構築
- **ヘッダー順序**: 接続テスト時に取得したCSVヘッダーの順序を維持
- **データ配置**: マッピング設定に基づいてContact Form 7の値を配置
- **空フィールド処理**: マッピングされていないフィールドは空文字で埋める
- **登録日処理**: 自動的に現在の日時を設定

### 3. API仕様

#### 3.1 使用するAPIコマンド

##### 3.1.1 downloadReaders（読者リスト取得）
```
!command=downloadReaders
!cid=<クライアントID>
!pass=<パスワード>
!includeValid=true
!includeInvalid=false
!includeError=false
```

**用途**: 
- 接続テスト
- CSVヘッダー取得
- 重複チェック

##### 3.1.2 updateReadersJob（読者登録ジョブ）
```
!command=updateReadersJob
!cid=<クライアントID>
!pass=<パスワード>
#email,名前
hanamoto@delightful.jp,ディライトフル
```

**用途**: 読者登録

#### 3.2 エラーハンドリング
- **CURLエラー**: 詳細なエラー情報をログに記録
- **HTTPエラー**: ステータスコードとレスポンス内容を記録
- **APIエラー**: レスポンス解析エラーの記録

### 4. セキュリティ仕様

#### 4.1 認証・認可
- **管理画面アクセス**: `manage_options`権限が必要
- **AJAX処理**: WordPressのnonce検証
- **データ検証**: 入力値のサニタイズ

#### 4.2 データ保護
- **パスワード**: WordPressオプションに暗号化せず保存（WordPressの標準的な方法）
- **API通信**: HTTPS通信
- **ログ**: デバッグ情報のみ記録

### 6. データベース仕様

#### 6.1 WordPressオプション
- `meruhai_endpoint_url`: エンドポイントURL
- `meruhai_login_id`: ログインID
- `meruhai_password`: パスワード
- `meruhai_field_mapping`: フィールドマッピング設定
- `meruhai_csv_headers`: CSVヘッダー

#### 6.2 データ形式
```php
// フィールドマッピング
array(
    'email' => 'your-email',
    '名前' => 'your-name',
    // ...
)

// CSVヘッダー
array(
    'email',
    '名前',
    '登録日',
    // ...
)
```

### 7. 動作フロー

#### 7.1 初期設定フロー
1. プラグイン有効化
2. 設定画面でAPI情報入力
3. 接続テスト実行
4. CSVヘッダー自動保存
5. フィールドマッピング設定
6. マッピング保存

#### 7.2 Contact Form 7送信フロー
1. Contact Form 7送信
2. `wpcf7_mail_sent`フック発火
3. 保存されたCSVヘッダー取得
4. マッピング設定に基づいてデータ構築
5. 重複チェック実行
6. める配くんAPIに読者登録
7. 結果をログに記録

### 8. エラー処理

#### 8.1 接続エラー
- **400 Bad Request**: リクエスト形式エラー
- **404 Not Found**: エンドポイントURLエラー
- **403 Forbidden**: 認証エラー
- **500 Internal Server Error**: サーバーエラー

#### 8.2 データエラー
- **重複エラー**: 既存読者との重複
- **マッピングエラー**: フィールドマッピング不備
- **CSVヘッダーエラー**: ヘッダー未保存

### 9. ログ仕様

#### 9.1 デバッグログ（WP_DEBUG有効時）
- プラグイン初期化
- AJAXハンドラー呼び出し
- APIリクエスト・レスポンス
- CSVヘッダー保存
- エラー詳細

#### 9.2 エラーログ
- 読者登録エラー
- API接続エラー
- 重複エラー

### 10. 設定画面仕様

#### 10.1 基本設定
- **エンドポイントURL**: URL形式検証
- **ログインID**: テキスト入力
- **パスワード**: パスワード入力

#### 10.2 フィールドマッピング
- **表示**: Contact Form 7フィールドとCSVヘッダーのプルダウン
- **保存**: AJAXでリアルタイム保存
- **検証**: 必須フィールドのチェック

#### 10.3 状態表示
- **接続状態**: 接続テスト結果
- **CSVヘッダー**: 保存されたヘッダー一覧
- **マッピング状態**: 設定済みマッピング数

### 11. パフォーマンス仕様

#### 11.1 処理時間
- **接続テスト**: 30秒タイムアウト
- **読者登録**: 非同期処理（ジョブ登録）
- **重複チェック**: 既存読者リストとの比較

#### 11.2 メモリ使用量
- **最小要件**: PHP 7.4以上
- **推奨**: PHP 8.0以上
- **メモリ**: 128MB以上

### 12. 互換性

#### 12.1 WordPress
- **対応バージョン**: 5.0以上
- **PHP**: 7.4以上
- **MySQL**: 5.6以上

#### 12.2 プラグイン
- **Contact Form 7**: 必須
- **その他**: 特に制限なし
